# デバッガリファレンス

エミュレータには、実行中のどの時点でもアクセスできるインタラクティブなデバッグ REPL が内蔵されています。別途デバッグビルドを用意することなく、Z80 および VDP の状態確認、ブレークポイント、ウォッチポイント、ステップ実行、トレース機能を提供します。

**⚙V9938** と記されたコマンドは MSX2 / V9938 マシン専用です。MSX1 / TMS9918A では例外を発生させず、わかりやすいエラーメッセージを表示します。

ソース：`msx/debugger/prompt.py`（REPL）、`msx/vdp/v9938.py` / `msx/vdp/v9938_renderer.py`（VDP）、`msx/mapper_tracer.py`（マッパートレース）

---

## デバッガへの入り方

### Ctrl+C

エミュレータを実行しているターミナル（または SDL ウィンドウ）で **Ctrl+C** を押します。エミュレーションが即座に一時停止し、現在の PC が逆アセンブルされ、REPL プロンプトが表示されます。

```
Debugger entered. Type 'c' to resume, 'q' to exit.
  PC=C010h  3E 01        LD A,01h
(msx-dbg cyc=720430 frm=12)
```

`cyc` は累積 Z80 T ステート数、`frm` は完了フレーム数です。表示されている画像は最後に描画されたフレームです。`c` で実行再開、`q` でエミュレータ終了です。REPL プロンプトが表示されている状態で再度 Ctrl+C を押すと、エミュレータが即座に終了します（`sys.exit(0)`）——入力のキャンセルやプロンプトへの復帰ではありません。

### 起動時にブレークポイント・ウォッチポイントを設定する

SDL ウィンドウが開く前に CLI フラグでブレークポイントやウォッチポイントを設定できます。

```bash
# 実行ブレークポイント — 最大 4 個、カンマ区切りの 16 進アドレス（0x プレフィックスなし）
python . path/to/game.rom --machine cbios_msx2_jp --break-point C000,D000

# ウォッチポイント — addr[,r|w|rw] ペアをカンマ区切りで指定；デフォルトモードは rw
python . path/to/game.rom --machine cbios_msx2_jp --watch-point ca4a,w,fedc,rw
```

有効なブレークポイントとウォッチポイントは起動時に stderr に出力されます。

## コマンドリファレンス

**`h`**、**`?`** — コマンド一覧を1行で表示する。未知のコマンドを入力した際にも自動的に表示されます。

### 実行制御

**`c`** — エミュレーションを再開する。

**`q`** — エミュレータを終了する（プロセスが終了コード 0 で終了）。

**`reset`** — CPU・PSG・SCC（存在する場合）・VDP・プライマリ/セカンダリスロットレジスタを完全にパワーオンリセットする。メモリ／VRAM の内容は保持されます。`c` と同様、実行後は即座にエミュレーションを再開します——リセット前の PC に確認すべきものは残っていないためです。

**`s [N]`** — Z80 命令を `N` ステップ実行する（10 進数；デフォルトは 1）。ステップ後にコンパクトなレジスタサマリと次の命令を表示します。

```
  PC=C012  AF=0100  BC=0010  DE=0000  HL=C100
  => C012: CD 40 C0     CALL C040h
```

**`g ADDR`** — 実行を再開し、PC が `ADDR`（16 進）に達するまで実行する。一度だけ有効な一時ブレークポイントで、ヒットするとクリアされます。`ba` のスロットは消費せず、永続ブレークポイントにも影響しません。

**`gf N`** — 実行を再開し、フレームカウンタ（プロンプトに表示される `frm=` の値）が10進数の `N` に達するまで実行する。命令単位ではなく描画フレームごとに1回だけ判定する一度限りのブレークポイントなので、`ba`/`g` のアクティブなブレークポイントのように実行速度を低下させません。`N` は現在のフレームカウンタより大きい必要があり、そうでない場合は `gf` はエラーを表示して再開しません。

**`so`** — ステップアウト：`so` を発行した時点の SP より SP が高くなるまで実行する（つまり現在のルーチンから戻るまで）。注意事項は「既知の制限」を参照してください。

---

### ブレークポイント

**`ba ADDR`** — `ADDR`（16 進、`0x` プレフィックスなし）に実行ブレークポイントを追加する。最大 4 個。PC がマッチするとエミュレーションが一時停止して REPL が開きます。

**`br ADDR`** — `ADDR` のブレークポイントを削除する。指定アドレスが存在しない場合はエラーを表示します。

**`bl`** — 有効なブレークポイントを一覧表示する。

#### クラッシュシグネチャ自動ブレーク

**`bh`** — 割り込み禁止状態での HALT 実行時ブレークを切り替える。CPU が `iff1 == 0` の状態で `HALT` を実行した瞬間に停止します——これは割り込みで起こされることのない、ゲームのハングを示すシグネチャです。立ち上がりエッジで 1 回だけ発火します。条件がまだ真の状態で再開しても再トリガされません。

**`bs [LOW HIGH | off]`** — SP が有効な RAM 範囲を外れた際にブレークする。

- `bs`（引数なし）：マシン RAM 設定から自動導出した範囲で有効化（MSX1 フラット RAM → トップオブメモリウィンドウ；MSX2 マッパー RAM → 全アドレス空間）。
- `bs LOW HIGH`：指定した 16 進の包括範囲で有効化。
- `bs off`：無効化。

SP が有効範囲を最初に外れた命令で停止します。立ち上がりエッジのみで発火します。

---

### ウォッチポイント

**`wa ADDR[,r|w|rw]`** — `ADDR` にウォッチポイントを追加する。モード：`r` = 読み出し時停止、`w` = 書き込み時停止、`rw` = 両方（デフォルト）。最大 4 個。

ヒット時に以下を表示します：

```
[WP] READ  CA4Ah = 02h  PC=C024h
```

または

```
[WP] WRITE CA4Ah = 05h  PC=C030h
```

表示後に REPL へ入ります。

**`wd ADDR`** — `ADDR` のウォッチポイントを削除する。

**`wl`** — 有効なウォッチポイントとそのモードを一覧表示する。

---

### CPU 状態

**`rc`** — Z80 のすべてのレジスタをダンプする。

```
AF=5A00  BC=0010  DE=0000  HL=C100  IX=F380  IY=FC40  SP=F3E0  PC=C010
  S=0  Z=1  H=0  P/V=0  N=0  C=0
```

**`da [ADDR]`** — `ADDR`（16 進）または現在の PC から 10 命令を逆アセンブルする。

```
  C010: 3E 01        LD A,01h
  C012: CD 40 C0     CALL C040h
  C015: 21 00 C1     LD HL,C100h
  ...
```

---

### メモリダンプ

**`dm ADDR [SIZE]`** — CPU アドレス空間の hex+ASCII ダンプ。両引数は 16 進；SIZE のデフォルトは 0x80（128 バイト）。

```
  F000: 00 01 02 03 04 05 06 07 08 09 0A 0B 0C 0D 0E 0F  ................
  F010: 41 42 43 44 45 46 47 48 49 4A 4B 4C 4D 4E 4F 50  ABCDEFGHIJKLMNOP
```

**`dv VADDR [SIZE]`** — VRAM の hex+ASCII ダンプ。両引数は 16 進；SIZE のデフォルトは 0x80。アドレスは VRAM サイズ内でラップします（TMS9918A は 0x3FFF；V9938 は 0x1FFFF）。アドレスプレフィックスの幅は自動調整されます。

**`dvf FILE`** — VRAM の全内容を生バイナリファイルとして書き出す。オフラインでの diff やリファレンスツールへの読み込みに便利です。

---

### VDP 状態

**`v`** — VDP のステータスサマリを表示する。

V9938 の場合（⚙V9938）：

```
  Screen : SCREEN5 (GRAPHIC4)
  VRAM   : Name=10000  Color=00000  Pat=00000  SprAttr=1FE00  SprPat=00000
  Disp   : EN=1 SPR=1 H=192 NTSC IL=0 SZ=0 MAG=0 FG=0F BG=00 IE0=1 IE1=0
  CMD    : ABRT LOG=IMP CE=0 TR=0  DX=0 DY=0 NX=0 NY=0
  Status : S#0=00 S#2=0C
```

TMS9918A の場合：

```
  Screen : SCREEN2 (GRAPHIC2)
  VRAM   : Name=1800  Color=2000  Pat=0000  SprAttr=1B00  SprPat=3800
  Disp   : EN=1 H=192 FG=0F BG=00 SZ=0 MAG=0
  Status : S#0=00
```

**`rv`** — VDP レジスタをダンプする。

V9938 の場合（⚙V9938）：R#0–R#27（表示レジスタ）と R#32–R#46（コマンドレジスタ）を 8 個ずつ表示します。

```
  R#0=00  R#1=72  R#2=1F  R#3=FF  R#4=3F  R#5=7F  R#6=07  R#7=00
  R#8=08  R#9=80  R#10=00  R#11=00  R#12=00  R#13=00  R#14=00  R#15=00
  ...
  R#32=00  R#33=00  R#34=00  R#35=00  R#36=00  R#37=00  R#38=00  R#39=00
  ...
```

TMS9918A の場合：R#0–R#7 を 1 行で表示します。

**`rp`** ⚙V9938 — パレットをダンプする。全 16 エントリを `#N=XXX(R,G,B)` の形式で表示します（XXX は 9 ビット GRB333 の生の値、R/G/B は 3 ビットチャンネル値）。

```
  #0=000(0,0,0)  #1=000(0,0,0)  #2=071(1,6,1)  #3=1FB(3,7,3)  ...
  #8=1C9(7,1,1)  #9=1DB(7,3,3)  #A=1B1(6,6,1)  #B=1B4(6,6,4)  ...
```

TMS9918A ではプログラマブルパレットがないため、わかりやすいエラーを表示します。

---

### スプライト

**`ds`** — 両 VDP タイプでスプライト描画のオン/オフを切り替える。オフの場合、背景レイヤーのみが描画されます。スプライト起因のアーティファクトの切り分けに便利です。次の切り替えまたはエミュレータ終了まで状態が維持されます。

---

### トレース

**`te`** ⚙V9938 — VDP レジスタ書き込みトレーサを有効化する。エミュレーション中のすべてのレジスタ書き込みを stdout に出力します。

```
CY=12450 FR=3 PC=C018  VDP_REG R#2=1F
CY=12460 FR=3 PC=C01C  VDP_REG R#5=7F
```

`te` は初回使用時に `_get_pc` と `_get_cycle` コールバックを自動的に配線します。`Tracer` が既にアタッチされている場合は再有効化します。

**`td`** — VDP レジスタトレースを無効化する。

**`ce`** — カートリッジマッパーのバンク切り替えトレーサを有効化する。バンクを変更するバンクレジスタへの書き込みごとに 1 行出力します。

```
CY=25000 FR=5 PC=5B2A  MAP_BANK win=1 03h->05h addr=6001h
```

`win` はマッパーウィンドウインデックス；`addr` はレジスタ書き込みのアドレスです。バンク切り替えマッパーが存在しない場合はメッセージを表示します。

**`cd`** — マッパーバンク切り替えトレースを無効化する。

---

### スロットインスペクタ

**`sl`** — アクティブなスロットテーブルを表示する。ページ（P0–P3）ごとに、アドレス範囲、プライマリスロット、セカンダリスロット、内容、バンク/セグメントを 1 行で示します。

```
  Page  Addr        Prim  Sec  Content                          Bank
  P0    0000-3FFF   0     -    ROM cbios_main_msx2.rom          -
  P1    4000-7FFF   1     -    Cartridge (mapped)               w0=b2@04000  w1=b0@00000
  P2    8000-BFFF   1     -    Cartridge (mapped)               w0=b3@06000  w1=b1@02000
  P3    C000-FFFF   3     2    RAM (mapper)                     seg=2
```

Bank 列の値：

- RAM マッパーページ：`seg=N`
- カートリッジ ROM マッパーページ：選択されたバンク番号と解決済みの ROM バイトオフセット範囲
- フラット ROM、BIOS、プレーン RAM：`-`

**`st`** — スロットツリーを表示する。プライマリスロット 0–3 を走査し、拡張スロットのサブスロット構造を完全に表示します。

```
  Primary 0  ROM cbios_main_msx2.rom  64 KB
  Primary 1  [cartridge slot]  Cartridge (mapped)  128 KB
  Primary 2  [cartridge slot]  (empty)
  Primary 3 [EXPANDED]  secondary-select(raw)=08h
    page-map: P0->3-0  P1->3-0  P2->3-2  P3->3-2
    3-0  ROM cbios_sub.rom  32 KB
    3-1  (empty)
    3-2  RAM (mapper)  128 KB
    3-3  (empty)
```

MSX1（`sub_slot_enabled = False`）では、スロット 3 は拡張なしのプライマリとして表示されます。

---

### フロッピーディスク

**`fdd1 [FILE|-]`**、**`fdd2 [FILE|-]`** — ドライブ A / B のディスクを表示・入れ替え・イジェクトする（フロッピーインターフェースを持つマシン、例：`hb_f1xd`）。

- 引数なしで現在のマウント状態（パス、または `empty`）を表示。
- パスを指定するとその `*.dsk` をマウント（cwd 相対または絶対パス；`~` を展開）。次の `c`（続行）で WD2793 がディスク交換ラインを通じて新しいディスクを認識します。
- `-` で現在のディスクをイジェクト。

エラー（ファイルなし、読み込み不可、ドライブなし）は現在のディスクを変えずに報告されます。フロッピーインターフェースを持たないマシンでは `this machine has no floppy interface` と表示されます。

```
(dbg) fdd1
fdd1: /path/to/game.dsk
(dbg) fdd1 disk2.dsk
fdd1: mounted disk2.dsk
(dbg) fdd1 -
fdd1: ejected
```

### スクリーンショット

**`ss`** — 現在の VDP 状態を描画し、`screenshot_YYYYMMDD_HHMMSS.png` を作業ディレクトリに保存する。内部フレームカウンタは変化しません；`ss` はエミュレーションの状態に影響を与えません。同じ停止時点で取得した `v` / `dv` / `rv` の出力と視覚的なスナップショットを対応させるのに役立ちます。

---

### ステートセーブ／ロード

**`sv [TITLE]`** — CPU・VDP・マッパー・RAM を含む完全なマシン状態を
`saves/states/<title>_YYYYMMDD_HHMMSS.state` に保存する。ソケット RPC の
`state.save` メソッド（[`docs/socket-rpc-mcp_ja.md`](socket-rpc-mcp_ja.md) 参照）と同じ形式・保存先です。引数なしの場合はデフォルトのタイトルを使用し、引数を指定した場合はそれがタイトルになります（ファイルパスではありません）。REPL は継続します。

**`ld FILE`** — 指定した `.state` ファイルからマシン状態を復元する。REPL は
継続し、自動的な再開は行わないため、`s` や `gf` でロードした状態を厳密に確認・進行できます。ファイルが存在しない場合や、`format_version` またはマシン種別が非互換なスナップショットの場合は、例外を送出せずエラーを表示します。

```
(msx-dbg cyc=720000 frm=1999) sv before-glitch
state saved: saves/states/before-glitch_20260101_120000.state
(msx-dbg cyc=720000 frm=1999) gf 2080
  Running to frame 2080 ...

... （フレーム 2080 に到達すると再度 REPL が開く） ...
(msx-dbg cyc=... frm=2080) ld saves/states/before-glitch_20260101_120000.state
  Loaded state from saves/states/before-glitch_20260101_120000.state
(msx-dbg cyc=720000 frm=1999) gf 2001
  Running to frame 2001 ...
```

## 起動時トレース（ヘッドレスキャプチャ）

以下のオプションは最初のフレームの前にトレースを設定するため、REPL に入らずに使用できます。

### VDP レジスタトレース

```bash
# VDP_REG 行を stdout に出力
python . path/to/game.rom --machine cbios_msx2_jp --vdp-trace

# ファイルにリダイレクト
python . path/to/game.rom --machine cbios_msx2_jp --vdp-trace --vdp-trace-out trace.log
```

出力フォーマット：`CY=.. FR=.. PC=..  VDP_REG R#N=XX`

### マッパーバンク切り替えトレース

```bash
python . path/to/game.rom --machine cbios_msx2_jp --mapper-trace
python . path/to/game.rom --machine cbios_msx2_jp --mapper-trace --mapper-trace-out banks.log
```

出力フォーマット：`CY=.. FR=.. PC=..  MAP_BANK win=W OLDh->NEWh addr=XXXXh`

フラット（バンク切り替えなし）マッパーのカートリッジの場合は、stderr にメッセージが出力され `MAP_BANK` 行は出力されません。

### ヘッドレス N フレーム実行

```bash
python . path/to/game.rom --machine cbios_msx2_jp --count-frame 300 --vdp-trace
```

`--count-frame N` は SDL ウィンドウを開かずに N フレームだけ実行して終了します。`--vdp-trace` や `--mapper-trace` と組み合わせることで、再現性のあるスクリプタブルなキャプチャが得られます。

## 既知の制限

**遅延レンダラ。** CPU が 1 フレーム分実行され、VDP コマンドが VRAM に即座に書き込まれ、フレーム末尾に 1 回のレンダリングパスが行われます。1 フレーム内でラスタに同期した VRAM 更新（ビームレースブリット、ダブルバッファタイトル画面）は忠実に再現されません。これはレンダラのバグではなく、アーキテクチャ上の制約です。

**コマンドタイミング。** VDP コマンドの完了（S#2 CE ビット）は近似サイクル予算で制御されており、サイクル精度モデルではありません。CE をポーリングするソフトウェアは動作しますが、正確なコマンド所要時間はハードウェアと異なります。

**`so` と `bs` は SP ヒューリスティックです。** `so` は SP が発行時の値を最初に上回った瞬間に停止します。スタックを切り替えるルーチン、プッシュより多くポップするルーチン、または再入されるルーチンでは早めや遅めに発火することがあります。正確な停止点が必要な場合は `g ADDR` を使ってください。`bs` は SP を静的なアドレス範囲と照合するため、スロット切り替えによって正当に RAM がリマップされるケースは考慮されません。
